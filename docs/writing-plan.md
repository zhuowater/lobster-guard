# 智能体安全手册写作计划

## 写作流水线

### Phase 1: 大纲定稿
- [ ] 子agent完成 v2 大纲 → `agent-security-handbook-v2.md`
- [ ] 主agent审核大纲质量，必要时调整

### Phase 2: 逐章写作
每章独立 spawn 子agent写作，完成后审核合并：
- [ ] 第1章 → `/tmp/handbook/ch01.md`
- [ ] 第2章 → `/tmp/handbook/ch02.md`
- [ ] ...
- [ ] 最后一章 → `/tmp/handbook/chNN.md`

### Phase 3: 合成与校对
- [ ] 合并所有章节为完整书稿 → `docs/agent-security-handbook-full.md`
- [ ] 第一轮校对：通顺性、逻辑连贯性
- [ ] 第二轮校对：术语统一、风格一致
- [ ] 第三轮校对：精简冗余、提升金句密度

### 写作要求
1. 零代码——算法用流程描述，不贴代码
2. 通俗化——科技爱好者能读懂
3. 故事驱动——每章开头用真实案例或类比引入
4. 有观点——不是教科书，是有锐度的思想写作
5. 科学严谨——理论部分数学公式后跟"人话翻译"
6. 目标：15-20万字
