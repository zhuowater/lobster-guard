## 4 端口架构

| 端口 | 功能 | 安全域 |
|------|------|--------|
| `:18443` | IM 入站代理 | IM 安全域 |
| `:18444` | IM 出站代理 | IM 安全域 |
| `:8445` | LLM 反向代理 | LLM 安全域 |
| `:9090` | Dashboard + API + Metrics | 管理层 |

## 数据流

### 完整模式（多上游）

```
消息平台 ──► :18443 (入站检测) ──► 路由表 ──► 上游容器池 ──► OpenClaw ×N
OpenClaw ──► :18444 (出站检测) ──► 消息平台 API
Agent    ──► :8445  (LLM 检测)  ──► LLM API (Anthropic/OpenAI)
运维团队 ──► :9090  (Dashboard)  ──► 管理 + 监控 + 态势大屏
```

### 单机模式（v1.0 兼容）

不配 `static_upstreams` 时自动退化：

```
消息平台 ──► :18443 ──► OpenClaw(:18790)
OpenClaw ──► :18444 ──► 消息平台 API
Agent    ──► :8445  ──► LLM API (Anthropic/OpenAI)
```

### Taint 全链路数据流 (v20.8.1+)

IM 入站标记 → LLM 传播 → SSE/非流式逆转 的完整链路：

```
┌──────────────────────────────────────────────────────────────────────┐
│                    Taint 全链路闭环                                    │
│                                                                      │
│  IM 入站 (:18443)                                                    │
│  ┌─────────────────┐                                                 │
│  │ 消息进入         │                                                 │
│  │ PII 检测（12 模式）│── 命中 ──→ taint_entries 表写入                  │
│  │ trace_id = T1   │          │  labels: [pii_phone, pii_id_card]   │
│  └────────┬────────┘          │                                      │
│           │                   ▼                                      │
│           │         SessionCorrelator 记录                            │
│           │         sender_id ↔ trace_id(T1) 关联                    │
│           ▼                                                          │
│  转发到 OpenClaw                                                      │
│           │                                                          │
│  LLM Proxy (:8445)                                                   │
│  ┌─────────────────┐                                                 │
│  │ Agent 发起 LLM 请求│                                               │
│  │ 查询 SessionCorrelator │                                          │
│  │ taintTraceID = T1 │ ← 优先使用 IM trace_id（非 LLM 自身 trace）     │
│  └────────┬────────┘                                                 │
│           │                                                          │
│           ├──── taint propagation ──→ taint_entries += "llm_request"  │
│           │                                                          │
│           ▼                                                          │
│  ┌─────────────────────────────────────────────┐                     │
│  │ 上游 LLM API 响应                             │                     │
│  ├──────────────────┬──────────────────────────┤                     │
│  │  非流式响应        │  SSE 流式响应              │                     │
│  │                  │                          │                     │
│  │  Reverse(T1)     │  stream flush to client  │                     │
│  │  ↓               │  stream ends →           │                     │
│  │  检查 taint labels│  Reverse(T1)             │                     │
```

### strip_prefix 路由 (v20.8.1+)

LLM Proxy 支持 `strip_prefix` 选项，在转发前去掉 `path_prefix`：

```
客户端请求                    strip_prefix=true            上游收到
─────────────               ─────────────────           ──────────
/qax/v1/chat/completions  →  去掉 /qax/v1/  前缀  →  /v1/chat/completions
                              target: aip.b.qianxin-inc.cn

/v1/chat/completions      →  strip_prefix=false     →  /v1/chat/completions
                              target: api.deepseek.com
```

用于将 OpenClaw 的 LLM 流量透明引入龙虾卫士审计，无需修改上游 API 路径。

### 安全层级
1. **L1 模式匹配** — AC自动机 + 正则 (μs级)
2. **L2 语义分析** — TF-IDF四维检测 (ms级)
3. **L3 行为分析** — 画像 + 基线 + 攻击链 (秒级)
4. **L4 密码学保证** — HMAC信封 + Merkle Tree (防篡改)
5. **L5 自进化** — 对抗变异 + 规则自动生成 (分钟级)
6. **L6 污染追踪** — IM↔LLM trace 关联 + SSE 流式逆转 (全链路)
