# 竞品学习报告 — 龙虾卫士 vs 奇安信产品线

> 2026-03-17 · 基于 NGSOC / 天眼 / 智慧防火墙 实际登录体验

## 研究产品

| # | 产品 | 版本 | 定位 | 学习角度 |
|---|------|------|------|---------|
| 1 | **NGSOC** (安全分析与管理系统) | V4.14.1 R4.15.1.SP4 | 安全运营中心 | 概览设计、AI 集成、风险评分 |

## 一、NGSOC — 安全运营中心

### 值得学习

**1. 综合风险评分 + 趋势线 = 一眼看懂安全态势**
- 左上角一个大数字 "22分"（低/中/高/危），配一条 30 天趋势线
- **龙虾卫士现状**：我们有用户风险分（v11.0），但没有**系统级综合风险分**
- **应借鉴**：首页加一个"安全健康分"——综合 IM 拦截率 + LLM 异常率 + 规则命中数 + Canary 泄露计算出 0-100 分

**2. "安全运营副驾驶" — AI 智能体直接嵌在首页**
- 右侧面板常驻 AI 对话入口："您好，我是安全智能体，您的安全运营副驾驶"
- 不是独立页面，而是**每个页面都可以呼出**
- **龙虾卫士现状**：纯被动 Dashboard，无 AI 辅助
- **应借鉴**：这是差异化方向——龙虾卫士本身就是 AI Agent 安全网关，为什么不让它自己也有 AI 能力？但需要想清楚：是展示还是实用

**3. 顶部信息密度极高**
- 一行内同时展示：产品名 + 功能区切换 + AI 标识 + 版本号 + 通知铃铛 + 用户名
- **龙虾卫士现状**：顶部只有面包屑 + 搜索 + 状态
- **应借鉴**：顶部可以加版本号 badge 和通知中心入口

**4. 自动刷新频率可选**
- "每5分钟刷新" 下拉框，用户可控
- **龙虾卫士现状**：硬编码刷新或无自动刷新
- **应借鉴**：加一个全局刷新频率设置

**5. 组织切换（多租户导航）**
- "默认组织" 下拉，一个 SOC 管多个组织
- **龙虾卫士现状**：单租户
- **暂不借鉴**：ROADMAP v14 有多租户计划，到时再学

### 不该学的
- 登录页的验证码设计——用户体验差，AI 也识别不了
- 顶部导航项太多会造成信息过载（NGSOC 的"监测中心"下面还有子导航）

## 二、天眼 — 威胁感知分析

### 值得学习

**1. 双导航模式：顶部功能域 + 左侧子页面**
- 顶部横向：监测中心 / 威胁感知 / 分析中心 / 响应处置 / 资产感知 / 报表报告 / 更多
- 左侧纵向：仪表板 / 系统运行监控 / 监测工作台 / 态势感知 / ATT&CK
- **龙虾卫士现状**：只有左侧导航，按 IM安全/LLM安全/系统 分组
- **应借鉴**：我们的分组思路是对的，但可以考虑顶部加功能域切换（当页面继续增长时）

**2. 态势感知大屏 — 全屏深色可视化**
- 独立全屏模式，深色背景 + 世界地图 + 攻击线 + 实时数据流
- **龙虾卫士现状**：没有全屏态势大屏
- **应借鉴**：v13 会话回放可以做类似的全屏可视化体验

**3. ATT&CK 攻击矩阵视图**
- 把告警映射到 MITRE ATT&CK 框架的攻击阶段（侦察→入侵→命令控制→横向渗透→目的执行→痕迹清理）
- 矩阵热力图显示每个战术/技术的告警数量
- **龙虾卫士现状**：LLM 规则有 category（prompt_injection/pii_leak/sensitive_topic/token_abuse），但没有框架映射
- **应借鉴**：可以做 OWASP LLM Top 10 矩阵视图——把我们的规则和告警映射到 LLM01-LLM10

**4. 仪表板自定义（视图配置管理）**
- 用户可以自定义仪表板布局：添加/删除/调整面板位置
- "恢复默认视图" + "仪表板配置管理" 按钮
- **龙虾卫士现状**：固定布局
- **应借鉴**：概览页的面板布局可配置化（长期目标）

**5. "重保模式" 一键切换**
- 顶部有"重保"按钮，点击进入护网/重大活动模式
- 可能改变告警策略、通知频率、大屏展示
- **龙虾卫士现状**：无类似概念
- **应借鉴**：可以做"严格模式"——一键切换所有规则从 warn 到 block，shadow mode 全部激活

**6. 告警攻击阶段矩阵**
- 行：告警类型（APT事件/攻击利用/恶意软件/拒绝服务/侦察/可疑活动...）
- 列：攻击阶段（侦察/入侵/命令控制/横向渗透/目的执行/痕迹清理）
- **龙虾卫士现状**：告警只有 category 分类
- **应借鉴**：可以做 LLM 攻击阶段矩阵（信息探测→提示注入→权限提升→数据窃取→持久化）

## 三、智慧防火墙 — 网络安全网关

### 值得学习

**1. 首页状态卡片行 — 一行看完全局健康**
- 8 个卡片一行排开：失陷主机 / 风险主机 / 威胁统计 / 告警信息 / CPU / 内存 / 磁盘 / 风扇
- 每个卡片：图标 + 标签 + 数字 + 子标签
- 颜色语义：绿=正常，黄=警告，红=危险
- **龙虾卫士现状**：4 个 StatCard，分布在不同页面
- **应借鉴**：首页（IM 概览）的 StatCard 可以增加系统健康指标（CPU/内存/磁盘），让运维一眼看到

**2. 顶部功能 Tab 式导航**
- 首页 / 分析中心 / 数据中心 / 处置中心 / 策略配置 / 对象配置 / 网络配置 / 系统配置
- 水平排列，每个 Tab 下有独立子页面体系
- **龙虾卫士现状**：左侧 sidebar 分组
- **对比**：sidebar 更适合我们当前的页面数量（13页），防火墙有 50+ 页面所以用顶部 Tab。暂不改

**3. 可折叠面板 + 拖拽布局**
- 每个面板有"关闭"和"缩小/展开"按钮
- 用户可以自定义首页展示哪些面板
- **龙虾卫士现状**：固定布局
- **应借鉴**：面板可折叠是低成本高回报的改进

**4. 接口实时流量图 — 实时数据流**
- ge1/ge2/ge3 接口的实时发送/接收流量（Kbps 级别）
- **龙虾卫士现状**：没有实时流量展示
- **应借鉴**：可以在监控页加代理请求的实时吞吐量图

**5. 风险等级五级配色**
- 红(5) / 橙(4) / 黄(3) / 蓝(2) / 绿(1)
- 比我们的 critical/high/medium/low 四级更细
- **龙虾卫士现状**：四级（critical/high/medium/low）
- **暂不改**：四级已经够用

**6. 多虚拟系统（VSYS）切换**
- 右上角 "root-vsys" 下拉，防火墙支持多虚拟系统
- **龙虾卫士现状**：单实例
- **暂不借鉴**：等 v14 多租户时再考虑

## 五、龙虾卫士行动计划

### 🔴 立即可做（高价值低成本）

1. **OWASP LLM Top 10 矩阵视图** — 把规则和告警映射到 LLM01-LLM10，类似天眼的 ATT&CK 矩阵
2. **首页综合安全健康分** — 一个 0-100 数字 + 趋势线，类似 NGSOC 的综合风险评分
3. **自动刷新频率设置** — 概览页加个刷新频率下拉框（30s/1m/5m/手动）
4. **"严格模式"一键切换** — 类似天眼的"重保模式"，一键把所有 warn→block、激活所有 shadow 规则

### 🟡 中期投入（有明显价值）

5. **面板可折叠** — 每个面板加折叠按钮，低成本高体验
6. **系统健康指标上首页** — CPU/内存/磁盘进度条，类似防火墙
7. **实时吞吐量图** — 监控页加代理请求的实时 QPS/延迟图
8. **通知中心** — 右上角铃铛，收集 Canary 泄露、预算超限、规则命中等告警

### 🟢 长期方向

9. **态势感知大屏** — 全屏深色可视化，攻击地图 + 实时数据流
10. **仪表板自定义** — 用户可拖拽/添加/删除面板
11. **AI 安全助手** — 类似 NGSOC 的副驾驶，但针对 LLM 安全场景

## 六、关键洞察

> **"安全产品的首页不是 Dashboard，是驾驶舱。"**

NGSOC 和防火墙的共同点：**首页一屏之内，运维人员能回答"系统现在安不安全？"这个问题。** 不需要点进子页面，不需要读表格，一个数字 + 几个色块就够了。

龙虾卫士的概览页目前更像"报表"而不是"驾驶舱"——有数字但缺少**一个总分**和**即时的安全判断**。

最值得立即做的一件事：**首页加综合安全健康分**。
