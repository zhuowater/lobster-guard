# lobster-guard Dashboard 演进路线

## 产品定位
从"开发者调试面板"进化为"企业安全运维中心"。
目标用户：安全运维人员、IT 管理员、部门安全负责人。

---

## 当前状态评审（v5.1 · 60KB 单 HTML）

### ✅ 做得好的
- 深色科技主题，辨识度高
- 信息密度合理，关键数据一目了然
- 单 HTML 零依赖，部署极简
- 规则管理、路由管理、审计日志功能完整

### 🟡 需要改进的（按用户体验优先级排序）

#### P0 — 基本可用性问题
1. **导航缺失**：11 个面板全部竖排，长页面需要反复滚动，找不到想要的面板
2. **响应式不足**：移动端 / 窄屏体验差（表格溢出、按钮挤压）
3. **数据刷新无反馈**：点击刷新后没有 loading 状态，不知道是在加载还是卡了
4. **错误提示弱**：API 失败只显示 "⚠️ 加载失败"，不告诉用户怎么解决
5. **版本号硬编码**：footer 写死 v5.0，实际是 v5.1（应从 API 动态获取）

#### P1 — 运维效率问题
6. **没有全局搜索**：想找某个用户的所有相关信息（路由、审计、风险）要在多个面板来回切换
7. **审计日志分页缺失**：一次加载全部日志，数据量大时卡顿
8. **时间范围选择不直观**：没有日期选择器，只能手动输入
9. **操作确认缺失**：解绑、清理、重置等危险操作没有二次确认
10. **批量操作反馈弱**：批量绑定后不知道成功了几条、失败了几条

#### P2 — 视觉与体验
11. **图表能力弱**：只有简单柱状图和环形图，缺少趋势线、饼图等
12. **颜色语义不统一**：有些地方 green=healthy，有些地方 green=pass，容易混淆
13. **空状态设计粗糙**：很多面板空数据时只显示一行文字，缺少引导
14. **表格没有排序功能**：审计日志、规则列表不能按列排序
15. **键盘快捷键缺失**：没有快速导航、快速搜索等快捷键

## 迭代计划

### Phase 1 — 导航与布局重构（v6.0-dashboard）
> 从"长页面"进化为"多页面应用"

**侧边栏导航**：
- 左侧固定宽度侧边栏（可折叠）
- 图标 + 文字导航项：概览 / 上游 / 路由 / 规则 / 审计 / 监控 / 设置
- 当前页高亮 + 面包屑
- 底部显示版本号和连接状态

**顶栏改进**：
- Token 移到设置页，顶栏只显示连接状态指示灯
- 全局搜索框（Ctrl+K 快捷键）：搜用户 ID / 规则名 / 审计内容
- 通知铃铛（实时 block 事件弹窗提醒）

**概览页（首页）**：
- 4 个核心大卡片：总请求 / 拦截数 / 拦截率 / 平均延迟
- 24h QPS 趋势图（折线，不是柱状）
- 最近 5 条攻击事件快速预览
- 5 维健康状态一行横排显示

**技术实现**：
- 基于 hash 路由（`#/overview`, `#/audit`, ...）
- 页面组件按需渲染，不一次性加载所有面板
- 侧边栏折叠状态存 localStorage

### Phase 2 — 表格与数据交互升级（v6.1-dashboard）
> 让数据可操作、可探索

**表格增强**：
- 列排序（点击表头切换升序/降序）
- 列宽可拖拽调整
- 列显隐控制（用户选择要看哪些列）
- 行选择（checkbox + 批量操作工具栏）
- 行点击展开详情（审计日志展开完整内容、规则展开完整 pattern）

**分页与虚拟滚动**：
- 审计日志分页（20/50/100 条每页 + 上一页/下一页）
- 路由表分页（当用户超过 100 条时）
- URL 参数同步（刷新页面保留筛选条件和页码）

**筛选增强**：
- 日期范围选择器（原生 datetime-local 输入）
- 筛选条件可保存为"预设"（常用筛选一键切换）
- 筛选结果计数显示

**操作安全**：
- 危险操作（解绑/清理/重置/删除）弹出确认模态框
- 模态框显示操作影响范围（"将解绑 3 个用户"）
- 操作成功/失败 toast 显示详细结果

### Phase 3 — 可视化升级（v6.2-dashboard）
> 从"数据展示"进化为"数据洞察"

**图表体系**：
- 趋势图：请求量 24h/7d 折线图（纯 CSS + SVG，不引入图表库）
- 饼图：拦截类型分布（injection / jailbreak / pii / ...）
- 热力图：按小时 × 星期的攻击频率热力图
- 延迟分布：P50/P95/P99 延迟柱状图
- 地理分布：如果有 IP 数据，按城市展示攻击来源（未来）

**实时监控改进**：
- QPS 改为平滑的 canvas 折线图（60 秒滑动窗口）
- 攻击事件流改为 timeline 样式（左侧时间轴 + 右侧卡片）
- 新增"攻击者画像"卡片：高风险用户列表 + 近期行为

**仪表盘自定义**：
- 用户可拖拽调整面板顺序
- 用户可选择在首页显示哪些卡片
- 布局偏好存 localStorage

### Phase 4 — 规则与策略管理可视化（v6.3-dashboard）
> 让非技术人员也能管理规则

**规则编辑器**：
- 在 Dashboard 直接创建/编辑/删除规则（而不是改 YAML 文件）
- 规则编辑表单：名称 / 类型(keyword/regex) / 模式 / 动作 / 优先级 / 分组
- 正则表达式实时测试（输入文本 → 高亮匹配结果）
- 规则启用/禁用开关（不删除，只是暂时停用）
- 规则导入/导出（YAML 上传下载）

**策略路由可视化**：
- 策略规则流程图：用户 → 匹配条件 → 目标上游（流程线连接）
- 策略优先级拖拽排序
- 策略测试工具升级：输入用户信息 → 显示会命中哪条策略 → 路由到哪个上游

**规则模板市场**：
- 浏览内置规则模板（通用/金融/医疗/政务）
- 一键启用/停用模板包
- 模板规则预览（展开查看每条规则的详细 pattern）

### Phase 5 — 运维工具箱（v6.4-dashboard）
> 一站式运维工具中心

**配置管理**：
- 在线查看当前运行配置（只读 + 语法高亮）
- 配置对比：当前配置 vs 默认配置，高亮差异
- 配置导出（下载当前完整配置）

**数据管理**：
- 备份管理面板：创建/下载/删除/恢复 一站式操作
- 归档管理面板：查看归档文件列表 + 下载 + 空间占用统计
- 数据库大小趋势图

**系统诊断**：
- 连通性测试：一键测试到上游的连接状态
- 规则性能分析：哪条规则检测最慢、正则超时统计
- 日志查看器：最近 N 条系统日志（结构化日志渲染）

**通知设置**：
- 在 Dashboard 配置告警 webhook 地址
- 告警规则：什么事件触发通知、通知频率
- 告警历史：最近发送的告警列表 + 送达状态

### Phase 6 — 多租户与权限（v6.5-dashboard）
> 企业级多角色访问控制

**角色系统**：
- 管理员：完全控制
- 运维人员：查看所有数据 + 操作规则和路由
- 观察者：只读（审计员、合规人员）
- Bot 管理员：只能管理自己负责的 Bot 的路由和规则

**登录认证**：
- 从 Token 直接输入 → 登录页面（用户名 + 密码 / SSO）
- 会话管理（登录态 + 超时自动登出）
- 操作审计（谁在什么时间做了什么操作）

## 技术约束

1. **单 HTML 文件** — 不引入 npm/webpack/React/Vue，保持零构建
2. **零外部 CDN** — 不加载任何外部 JS/CSS/字体
3. **纯 CSS + 原生 JS** — 图表用 SVG + CSS 手绘，不引入 chart 库
4. **go:embed 嵌入** — 编译进二进制，部署零配置
5. **渐进增强** — 每个 Phase 独立可用，不依赖后续 Phase

## 实施节奏

每个 Phase 对应一个版本，Phase 内的功能可以拆成多个 commit 逐步实现。
优先级：Phase 1 > Phase 2 > Phase 3 > Phase 4 > Phase 5 > Phase 6。
Phase 1-3 是核心体验提升，Phase 4-6 是企业级增强。
