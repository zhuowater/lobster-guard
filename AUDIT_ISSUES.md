# Dashboard 全站审查问题清单

## 🔴 P0 - 功能缺失/数据断裂

### 1. 驾驶舱无治理引擎数据
- Overview StatCard只有 总请求/拦截数 两个基础指标
- 缺少: IFC违规数、计划偏差数、Capability拒绝数、反事实验证数
- 安全健康分(85)没考虑v23-v26引擎数据
- **修复**: 子agent正在处理

### 2. 威胁中心不关联治理引擎事件
- 审计日志页面拦截数=0，但首页显示192——数据源不一致(审计页只看block action，首页看stats)
- 用户画像(UserProfiles)的192告警不包含IFC violation
- 攻击链分析不包含IFC/Capability违规作为攻击节点
- **修复**: 需要后端在审计日志写入时加IFC/Capability事件

### 3. 侧边栏治理引擎组滚动不到
- 策略引擎tab下4个分组，治理引擎在第3组，但侧边栏没滚动条
- 当浏览器高度不够时，治理引擎和系统策略组被截断看不到
- **修复**: Sidebar添加overflow-y: auto

## 🟡 P1 - UX/交互问题

### 4. 图标重复，辨识度低
- shield: 入站规则 / 信息流控制 / (top nav)
- git-branch: 路由策略 / 路径策略 / 执行计划
- microscope: 反事实验证 / 语义检测
- wrench: 工具策略 / 运维工具
- **修复**: 每个页面用独特图标

### 5. 治理引擎页面没有中文标题
- PathPolicy: "Path Policy Engine" 英文
- Counterfactual: "反事实验证引擎" ✓
- Capability: "Capability Engine" 英文
- PlanCompiler: "执行计划编译器" ✓
- IFC: "IFC 信息流控制" ✓
- **修复**: 统一中文

### 6. StatCard布局不一致
- Overview: 2列
- 治理引擎各页: 2列×2行
- Capability: 4列×1行
- **修复**: 统一布局规范

### 7. 页面之间缺少跳转链接
- IFC违规记录没有"查看用户画像"链接
- 攻击链没有"查看相关IFC事件"链接
- 计划偏差没有"查看原始审计日志"链接
- **修复**: 关键数据点加router-link

### 8. 空状态引导不统一
- 有的页面空数据显示"0"，有的显示空表格
- 缺少"暂无数据"引导文案和操作建议
- **修复**: 统一EmptyState组件

## 🟢 P2 - UI打磨

### 9. 面包屑不准确
- 所有页面面包屑都是"龙虾卫士 / 页面名"
- 应该是"龙虾卫士 / Tab名 / 分组名 / 页面名"

### 10. 颜色语义不一致
- StatCard颜色含义不统一：有的红=危险，有的红=拦截(好事)
- IFC页面: violations用红色 ✓, hidden/DOE用紫色(语义不明)
- **修复**: 建立颜色语义规范

### 11. 响应式/移动端
- 侧边栏在窄屏下无法折叠（折叠按钮被截断）
- StatCard在小屏幕挤在一起

### 12. 页脚/版权信息缺失
- 无任何页脚信息
- 建议加: "© 2026 Lobster Guard v26.2 | Powered by 龙虾卫士"
